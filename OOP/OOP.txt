Objected-oriented programming (OOP) in Python helps you structure your code by grouping related
data and behaviors into objects. You star by defining classes, which act as blueprints, and then create objects from them,
OOP simplifies modeling real-world concepts in your programs and enables you to build systens that are more reusable and scalable.

Object-oriented programming in python involves creating classes as blueprints for objecrts. These objects contain data and methods needed
to manipulate that data.

The four key concepts of OOP in python are encapsulation, inheritance, abstraction, and polymorphism.

we can create an object in python by instantiating a class, which involves calling the class name followed by parentheses.

Class inheritance in python allows a class to inherit attributes and methods from another class,
known  as the parent class.

Use super()in python to call a method from the parent class, allowing you to extend or modify inherited behavior.


1 - Encapsulation allows you to bundle data (attributes) and behaviors (methods) within a class to create a cohesive unit
By defining methods to control acess to attributes and its modification, encapsulation helps maintain data integrity and 
promotes modulare, secure code.

2 - Inheritance enables the creation of hierarchical relationships between classes, allowing a subclass to inherit attributes and 
methods from  a parent class. This promotes code reuse and reduces duplication.

3 - Abstraction focuses on hiding implementation details and exposing only the essential functionality of an object. By enforcing a consistent
interface, abstraction simplifies interactions with objects, allowing developers to focus on what an object does rather than how it achieves its
functionlity 

4 - Polymorphism allows you to treat objects of different types ans instances of the same base type, as long as they implement a common interface
or behavior. Python's duck typing, make it expecially suited for polymorphism, as it allows you to acces attributes and methods on objects without
needing to worry about their actual class.
